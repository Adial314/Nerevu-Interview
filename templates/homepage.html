Introduction

This API was designed for a technical interview with Nerevu Group in 2019. It returns dates for upcoming holidays by scraping the Time and Date API into a CSV and then returning subsets of this master file as JSON responses. Please read below to learn about how this API works.

To read the development documentation for this API, please visit http://127.0.0.1:5000/docs.

Tutorial

Within this section I will describe how to run and interact with the holiday API.

Starting the API

To run this API you will need a python environment with the requisite libraries, listed below, and a browser to interact with the API. Most of these modules and libraries are installed by default with the Anaconda Python distribution but I have included links to install these libraries for your convenience:

  • Flask
  • Numpy
  • Pandas
  • Datetime
  • Fuzzywuzzy
  • Re
  • Pathlib

Create a new Anaconda environment and install the required libraries. Keeping your environment active, navigate to the directory of the API and run python API.py. This will start running the API on your local machine at http://127.0.0.1:5000/. Open your browser of choice to http://localhost:5000/ and you will arrive on the home page of the API.

Calling Endpoints

Thus far there is only one endpoint available within the API. To access this endpoint, go to /api/holiday. This will return the next ten upcoming holidays with respect to the time of the query.

Using the record_num argument, users can select how many holidays they would like to receive from a single query call. This argument can be passed like so: /api/holiday?record_count=.... The API will return ten holidays if num is not specified.

To filter by the type of holiday, users can pass the holiday_type argument through the API like so /api/holiday?holiday_type="...". Possible filter options include Federal, State, Observance, Jewish, Christian, etc.

Users may choose to combine several arguments in a single query like so: /api/holiday?record_count=...&holiday_type="...".. It is important to note that using multiple arguments may not return the number of holidays that were expected if the holiday type occurs infrequently or if there are no remaining instances of that holiday within the time period of study.

Version History

This API is new, experimental, and is designed mostly to demonstrate several technical proficiencies. With this context in mind, it is unlikely that it will become a fully fledged API with more complicated features. However, I can see an immediate use for it so I will treat the documentation as if it were the start of something big. Ergo, I have developed the following section to detail the version history of this API as well as to plan for features to come in subsequent versions.

V0.1

This is the first beta version of the API. It has a single endpoint holidays/ with arguments holiday_type and record_count available to the user.

V0.2 (upcoming)

Next I will add a screen scraping ability which updates the database on a regular basis by reading from another holiday table.

V0.3 (upcoming)

In the upcoming beta update, the holidays API will have the ability to filter by additional columns such as weekday and name.

A help endpoint will be developed to assist users to sort the available holiday type options.
